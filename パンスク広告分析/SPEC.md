# SPEC.md — 分析仕様・認識合わせ

## データ構造の理解

### シート構成
| シート名 | 内容 |
|---|---|
| ローデータ | Meta広告管理画面からのエクスポート（97行） |
| 成果の良いCR | CV獲得クリエイティブ（10本、合計59CV） |
| 成果の悪いCR | CV0のクリエイティブ（IMP/クリックあり） |

### 主要指標の定義
| 指標 | 定義 | 備考 |
|---|---|---|
| インプレッション | 広告が表示された回数 | |
| リーチ | 広告が表示されたユニークユーザー数 | |
| フリークエンシー | 1ユーザーへの平均表示回数 = IMP/リーチ | |
| リンクのクリック | 広告からのリンクへのクリック数 | |
| CTR | クリック率 = クリック/IMP | |
| 結果 | CV数（購入 or 登録） | |
| 結果タイプ | ウェブサイト購入 or ウェブサイトの登録完了 | |
| 結果の単価 | CPA = 消化金額/CV数 | |
| 消化金額 | 広告費用（円） | |
| CPM | 1000IMP当たりコスト = 消化金額/IMP×1000 | 計算値 |
| CVR | CV率 = CV数/クリック数 | 計算値 |

### キャンペーン構造
| キャンペーン名 | 広告セット | 目的 |
|---|---|---|
| ASC キャンペーン | ASC 広告セット | メイン配信（自動最適化） |
| ASC キャンペーン _検証 | ASC 広告セット_検証 | テスト検証用 |
| INTキャンペーン | 女性_新CR | インタレスト（興味関心）ターゲティング |

### クリエイティブ種別の判定ルール
- **動画**: 広告名に「動画」が含まれる（動画_04〜動画_09）
- **カルーセル**: 広告名に「カルーセル」が含まれる（2405_カルーセル01/02）
- **画像**: それ以外（no○○ 形式）

## 重要な前提・注意点

1. **結果(CV)がNaNの行**: CV未獲得（0として扱う）
2. **同一CR名が複数キャンペーンに存在**: 例えばno54_bはASCとINT両方に配信
3. **配信ステータス**: active（配信中）とnot_delivering（停止中）が混在
4. **結果の単価**: CV=1の場合は消化金額がそのままCPA。CV複数の場合は除算済み
5. **INTキャンペーン**: ほぼCVなし（no44の登録完了1件のみ）
6. **フリークエンシー**: 全体的に1〜1.7程度と低め（広告疲れは発生していない）

## 分析スコープ外

- 日別・週別のトレンド分析（ローデータに日次データなし）
- オーディエンス詳細分析（年齢/性別別データなし）
- クリエイティブの画像内容評価（画像ファイルなし）
