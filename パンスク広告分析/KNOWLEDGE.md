# KNOWLEDGE.md — 学び・ナレッジ

## データ特性の学び

### Meta広告エクスポートデータの特徴
- **結果タイプがNaN** = CV未獲得（0扱い）。NULL ≠ エラーなので注意
- **同一CR名が複数行に存在** = キャンペーンをまたいで同じCRを使い回すパターンがある
- **結果の単価** = CV複数獲得の場合は既に除算されている（合計金額/CV数）
- **フリークエンシー** = リーチ数が少ない広告は1.0に近くなりやすい（1回しか見ていない）

### Excelシートの読み方
- `成果の良いCR`・`成果の悪いCR`シートは、ヘッダーが2行目から始まっている（`header=1`で読む）
- 1行目にデータ期間の注記が入っているため、`header=0`で読むと列名がズレる

## BigQuery × GA4 データ基盤の学び（2026/03/19調査）

### インフラ構成の確認結果
- **サイト構成**: TOPページ=WordPress、LP=STUDIO、会員・マイページ=スクラッチ開発
- **Firestore → BQ**: `firestore_export` データセットに `pansuku_accounts_raw_latest` として連携済み
- **GA4 → BQ**: `analytics_312681459` が正解（meta/ig流入あり）。`analytics_250831881` はdirectのみで別サイト用
- **3つのGA4データセット**: `analytics_250831881`（JP、別用途）、`analytics_312681459`（メイン）、`analytics_438269555`（用途不明）

### Firestoreのaccountsスキーマ（重要フィールド）
```
gaClientId          GA4のuser_pseudo_idと一致 → JOINキー
plan                oneMonth / twoMonths / twoWeeks
signedUpAt          登録日時
registerCompleted   決済情報入力完了でtrue（登録フォーム完了ではない）
registerCompletedAt 決済完了日時
suspended           休止中フラグ
deleted             解約フラグ
leaveReason         解約理由
nextShipAfter       次回発送予定日
subscOrderChangedLogs プラン変更履歴
```
- **UTM系フィールドは存在しない** → Firestoreに直接UTMは保存されていない

### GA4 × Firestore JOIN の方法
```sql
JOIN ON JSON_VALUE(a.data, '$.gaClientId') = g.user_pseudo_id
```
- `gaClientId`（Firestore）と `user_pseudo_id`（GA4）は同一形式で結合可能
- JOIN対象データセット: `pansuku-app.analytics_312681459.events_*`

### イベント定義
| GA4イベント名 | タイミング |
|---|---|
| `サービス登録開始` | 登録フォームを開いた |
| `register_complete` | 登録フォーム完了（決済前） |
| `registerCompleted = true`（Firestore） | **決済情報入力完了**（実質有料会員化） |

### クリエイティブ単位のLTV追跡に必要な残り作業
- **代理店依頼**: Meta広告のURLパラメータに `utm_content={{ad.name}}` を追加
- これにより GA4の `traffic_source.name` にCR名が入り、CR単位の継続率・解約率が計測可能になる

### 休会・復帰分析の限界（2026/03/19調査）
- `false → true`（休会イベント）がBQ上で0件 = 休会履歴がCDCに記録されていない
- `suspended = true` のユーザーは13名存在するが、いつ・どの経路で休会したか追えない
- `suspendedHistories` フィールドも全件NULL
- **原因の仮説**: ①CDCエクスポート開始前の休会が初期状態として引き継がれた、②休会処理がCDCを経由しない仕組み（Cloud Functions等）
- **エンジニア確認事項**: 休会処理がFirestoreを直接update()しているか / BQ CDC開始日はいつか / 別の休会ログテーブルが存在するか

### BQクエリのTips
- BigQueryで日本語カラム名（AS 登録者数）はエラーになる → 英語エイリアスを使う
- `events_*` ワイルドカードテーブルはバッククォートで全体を囲む: `` `project.dataset.events_*` ``
- `collected_traffic_source` は新しいGA4エクスポートのみ。古い形式では `traffic_source` を使う

## 解約分析の結果（2026/03/19調査）

### leaveReason（解約理由）の傾向
- **断トツ1位：価格・コスト系**（「高い」「値段が高い」「料金が高い」「経済的理由」等の表記ゆれ含め推定50%超）
- **2位：無記入・空欄**（NULL + 空文字で約22%。"あ"や"。"等の形式的入力も含む）
- **3位：クーポン使えない系**（約260件）→ **解約まで平均2〜18日**と超短期。初回クーポン目的の入会→使えないと知って即解約のパターン
- **その他**: 必要なくなった / 好みのパンではなかった / 美味しくない / 食べきれない / ダイエット / 体調不良 / 引越し / 冷凍庫に入らない
